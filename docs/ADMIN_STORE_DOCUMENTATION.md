# توثيق إدارة متجر البيك

هذا الدليل يشرح آلية عمل لوحة الإدارة والمتجر من وجهة نظر الأدمن: وظيفة كل إدارة، معنى الحقول عند الإضافة أو التعديل، وكيف تنعكس البيانات على واجهة المتجر والطلبات.

## معلومات الدخول

رابط لوحة الإدارة:

```text
/admin/login
```

حسابات التجربة:

```text
Super Admin: admin@albaikstore.local / password
Admin:       admin@qr.com / password
Manager:     manager@qr.com / password
Customer:    customer@qr.com / password
Wholesale:   wholesale@qr.com / password
```

## قواعد عامة قبل البدء

- الحقول متعددة اللغات تظهر على شكل تبويبات مثل العربية والإنجليزية.
- الحقول النصية المترجمة مثل اسم المنتج والوصف يجب تعبئتها لكل لغة فعالة.
- الحقول الرقمية مثل السعر والمخزون والوزن لا تكرر حسب اللغة، لأنها قيم تجارية واحدة مشتركة.
- الحقول التي فيها `Status` أو `Is Active` تتحكم بظهور السجل أو استخدامه في المتجر.
- الصور ترفع إلى `storage/app/public` وتظهر عبر رابط `storage`، لذلك يجب تشغيل:

```bash
php artisan storage:link
```

## إدارة الهوية والمظهر

المسار:

```text
Settings > Site Appearance
```

هذه الصفحة تتحكم بهوية المتجر العامة، الألوان، الشعار، بيانات التواصل، وروابط الشركة.

### Identity

- `Store name Arabic`: اسم المتجر بالعربية.
- `Store name English`: اسم المتجر بالإنجليزية.
- `Tagline Arabic`: عبارة قصيرة تظهر تحت اسم المتجر بالعربية.
- `Tagline English`: عبارة قصيرة تظهر تحت اسم المتجر بالإنجليزية.
- `Short description Arabic`: وصف مختصر يظهر في الفوتر وبعض أقسام المتجر.
- `Short description English`: نفس الوصف باللغة الإنجليزية.
- `Logo`: شعار المتجر.
- `Favicon`: أيقونة التبويب في المتصفح.
- `Default product image`: صورة افتراضية تستخدم عند عدم وجود صورة منتج.

### Colors

تتحكم هذه الحقول بألوان واجهة المتجر:

- `Primary color`: اللون الأساسي للأزرار والعناصر المهمة.
- `Primary hover color`: لون التحويم على الأزرار.
- `Accent color`: لون مساعد للعروض والتنبيهات.
- `Topbar color`: لون الشريط العلوي إن تم استخدامه.
- `Header bg color`: خلفية الهيدر.
- `Nav bg color`: خلفية شريط التنقل.
- `Body bg color`: خلفية الصفحات.
- `Surface color`: لون البطاقات والأسطح.
- `Surface tint color`: لون خفيف للخلفيات المساعدة.
- `Text color`: لون النص الأساسي.
- `Muted text color`: لون النص الثانوي.
- `Border color`: لون الحدود.
- `Hero overlay start / end`: ألوان التدرج فوق صور السلايدر.

### Contact

- `Email`: بريد الدعم.
- `Phone`: رقم الهاتف.
- `WhatsApp`: رقم واتساب الشركة.
- `Address Arabic / English`: عنوان الشركة حسب اللغة.
- `Working hours Arabic / English`: ساعات العمل.
- `Map URL`: رابط خريطة Google أو أي رابط موقع.

### Social Links

روابط الشبكات الاجتماعية مثل Facebook وInstagram وYouTube وTikTok وX وLinkedIn. تظهر في الفوتر أو صفحات التواصل.

## إدارة اللغات

المسار:

```text
Settings > Languages
```

تتحكم هذه الإدارة باللغات المتاحة في المتجر ولوحة الإدارة.

- `Name`: اسم اللغة مثل Arabic.
- `Code`: كود اللغة مثل `ar` أو `en`. هذا الكود يستخدم داخل JSON للترجمات.
- `Direction`: اتجاه اللغة، `rtl` للعربية و`ltr` للإنجليزية.
- `Flag`: رمز أو علم اختياري للعرض.
- `Sort order`: ترتيب اللغة في القوائم.
- `Is default`: يجعل اللغة هي الافتراضية.
- `Is active`: يفعّل اللغة في المتجر ولوحة الإدخال.

مهم: عند إضافة لغة جديدة ستظهر تبويباتها في نماذج المنتجات والتصنيفات والبراندات والبانرات.

## الكتالوج

### المنتجات

المسار:

```text
Catalog > Products
```

المنتج هو العنصر الأساسي في المتجر. يظهر في صفحة المنتجات، التفاصيل، العروض، السلة، والطلبات.

#### Product Information

داخل تبويب كل لغة:

- `Name`: اسم المنتج كما يظهر في المتجر.
- `Short description`: وصف مختصر يظهر في البطاقات والتفاصيل.
- `Description`: وصف تفصيلي في صفحة المنتج.
- `SEO Title`: عنوان SEO للصفحة.
- `SEO Description`: وصف SEO لمحركات البحث.

حقول مشتركة بين اللغات:

- `Slug`: الرابط المختصر للمنتج. مثال: `iphone-15-pro-128gb`. يجب أن يكون فريداً ويستخدم في رابط صفحة المنتج.
- `SKU`: كود داخلي للمخزون والطلبات. لا يظهر عادة للعميل، لكنه مهم للإدارة والمستودع.
- `Barcode`: باركود المنتج. هو نص وليس رقم حتى لا تضيع الأصفار الأولى أو الرموز.
- `Brand`: العلامة التجارية.
- `Supplier`: المورد.
- `Category`: التصنيف.
- `Tags`: وسوم مثل هواتف، مميز، جملة.
- `Retail Price`: سعر التجزئة العادي.
- `Stock Quantity`: مخزون المنتج الأساسي إذا لم تستخدم متغيرات.
- `Low stock threshold`: الحد الذي يعتبر عنده المنتج منخفض المخزون.
- `Weight`: وزن المنتج، يستخدم في قواعد الشحن حسب الوزن.
- `Is Featured`: يجعل المنتج مميزاً في الواجهة.
- `Status`: تفعيل أو إخفاء المنتج.

#### Variants

المتغيرات تستخدم عندما يكون للمنتج أكثر من لون أو حجم أو ذاكرة.

- `SKU`: كود المتغير، مثل كود لون أو سعة معينة.
- `Barcode`: باركود المتغير إن وجد.
- `Attributes`: خصائص المتغير بصيغة مفتاح وقيمة. مثال: `color = Black`, `storage = 128GB`.
- `Stock`: مخزون هذا المتغير.
- `Reserved Stock`: مخزون محجوز للطلبات، يظهر للقراءة ولا يعدل يدوياً عادة.
- `Low stock threshold`: حد انخفاض مخزون المتغير.
- `Price`: سعر خاص للمتغير إن أردت تمييزه.

#### Images

- `Path`: صورة المنتج.
- `Alt text`: وصف بديل للصورة.
- `Is primary`: يجعل الصورة هي الرئيسية.

#### Media

- `Video URL`: رابط فيديو للمنتج.

#### Price Tiers

تظهر داخل صفحة تعديل المنتج كجدول علاقة.

- `Type`: نوع الشريحة، `retail` أو `wholesale`.
- `Minimum Quantity`: أقل كمية لتطبيق السعر.
- `Price`: السعر عند تحقق الكمية.
- `Sort Order`: ترتيب الشريحة.
- `Active`: تفعيل أو إيقاف الشريحة.

آلية السعر:

- الزائر والعميل العادي يشاهدان سعر التجزئة.
- تاجر الجملة المعتمد يرى شرائح الجملة.
- عند إضافة المنتج للسلة أو إنشاء الطلب يحسب السيرفر السعر النهائي، ولا يعتمد على JavaScript.

### العلامات التجارية

المسار:

```text
Catalog > Brands
```

- `Name`: اسم البراند في كل لغة.
- `Description`: وصف البراند في كل لغة.
- `Slug`: الرابط المختصر لصفحة البراند.
- `Logo`: صورة شعار البراند.
- `Status`: تفعيل أو إخفاء البراند.

تظهر البراندات في صفحة العلامات التجارية وتستخدم كفلتر في صفحة المنتجات.

### التصنيفات

المسار:

```text
Catalog > Categories
```

- `Name`: اسم التصنيف في كل لغة.
- `Description`: وصف التصنيف.
- `Slug`: الرابط المختصر.
- `Parent Category`: التصنيف الأب لإنشاء تصنيف فرعي.
- `Status`: تفعيل أو إخفاء التصنيف.

التصنيف يستخدم لتنظيم المنتجات والفلاتر وصفحات `/categories`.

### الموردون

المسار:

```text
Catalog > Suppliers
```

- `Name`: اسم المورد.
- `Slug`: رابط/كود مختصر للمورد.
- `Email`: بريد المورد.
- `Phone`: هاتف المورد.
- `Address`: عنوان المورد.
- `Is active`: هل المورد فعال.

المورد لا يظهر عادة للزائر، لكنه مهم للإدارة وسلسلة التوريد.

## المبيعات والطلبات

### الطلبات

المسار:

```text
Sales > Orders
```

الطلب ينشأ من checkout بعد اختيار العنوان وطريقة الشحن والدفع.

#### الحقول

- `Status`: حالة الطلب.
- `Customer Shipping Information`: بيانات العميل والشحن، تظهر للقراءة.
- `Tracking number`: رقم تتبع الشحنة.
- `Notes`: ملاحظات على الطلب.
- `Timeline`: سجل تاريخي لحركات الطلب.

#### الحالات

- `Pending`: طلب جديد بانتظار الدفع أو المراجعة.
- `Paid`: تم تأكيد الدفع.
- `Processing`: قيد التحضير.
- `Shipped`: تم الشحن.
- `Delivered`: تم التسليم.
- `Cancelled`: ملغى.
- `Refunded`: مسترد.

#### الإجراءات

- `Invoice`: تحميل فاتورة PDF.
- `Mark Paid`: تأكيد الدفع.
- `Process`: نقل الطلب للمعالجة.
- `Ship`: إدخال رقم التتبع ونقل الطلب للشحن.
- `Deliver`: إنهاء الطلب كمسلم.
- `Cancel`: إلغاء الطلب.

ملاحظة مهمة: عند إنشاء الطلب يتم حفظ سعر كل منتج داخل `order_items` حتى لو تغير سعر المنتج لاحقاً.

### المدفوعات

المسار:

```text
Sales > Payments
```

- `Order`: الطلب المرتبط.
- `Payment method`: طريقة الدفع.
- `Driver`: مشغل الدفع مثل manual أو cod.
- `Status`: حالة الدفع.
- `Amount`: المبلغ.
- `Transaction reference`: مرجع العملية.
- `Receipt image`: صورة إشعار الدفع.
- `Submitted at`: وقت رفع الإشعار.
- `Payload`: بيانات إضافية تقنية.

تستخدم الإدارة لمراجعة التحويلات اليدوية وتأكيد الدفع.

### طرق الدفع

المسار:

```text
Sales > Payment Methods
```

- `Name`: اسم طريقة الدفع حسب اللغة.
- `Description`: تعليمات أو وصف طريقة الدفع.
- `Slug`: كود فريد للطريقة.
- `Type`: نوع الدفع، مثل `cod`, `bank_transfer`, `manual`.
- `Image`: شعار طريقة الدفع.
- `Wallet URL`: رابط محفظة، رقم حساب، IBAN، أو معلومات الدفع.
- `Barcode image`: QR أو باركود الدفع.
- `Settings`: إعدادات إضافية مثل اسم البنك أو رقم الحساب.
- `Fee`: رسوم إضافية تضاف للطلب.
- `Is active`: تفعيل الطريقة للعميل.

آلية الدفع اليدوي:

1. العميل يختار طريقة الدفع.
2. تظهر معلومات المحفظة أو البنك.
3. يرفع صورة إشعار الدفع.
4. الطلب يدخل مراجعة.
5. الأدمن يؤكد الدفع من المدفوعات أو الطلبات.

## الشحن

### طرق الشحن

المسار:

```text
Sales > Shipping Methods
```

- `Name`: اسم طريقة الشحن حسب اللغة.
- `Description`: وصف أو تعليمات الشحن.
- `Slug`: كود فريد.
- `Zone`: وصف عام للمنطقة.
- `Type`: `flat_rate` أو `rule_based`.
- `Cost`: تكلفة افتراضية.
- `Free shipping minimum`: إذا تجاوز الطلب هذا المبلغ يصبح الشحن مجانياً.
- `Rules`: خريطة إعدادات إضافية مستقبلية.
- `Is active`: تفعيل طريقة الشحن.

### مناطق الشحن

المسار:

```text
Sales > Shipping Zones
```

- `Name`: اسم المنطقة.
- `Country`: البلد.
- `City`: المدينة أو المحافظة.
- `Town`: البلدة أو الحي.
- `Is active`: تفعيل المنطقة.

مثال: سوريا > دمشق > المزة.

### قواعد الشحن

المسار:

```text
Sales > Shipping Rules
```

- `Shipping method`: طريقة الشحن التي تتبع لها القاعدة.
- `Shipping zone`: المنطقة التي تطبق عليها.
- `Calculation type`: نوع الحساب.
- `Min / Max Quantity`: شروط حسب عدد المنتجات.
- `Min / Max Weight`: شروط حسب الوزن.
- `Min Subtotal`: حد أدنى لقيمة الطلب.
- `Cost`: التكلفة الأساسية.
- `Cost per kg`: تكلفة كل كيلو عند الحساب بالوزن.
- `Is free`: يجعل الشحن مجانياً.
- `Is active`: تفعيل القاعدة.

عند checkout يبحث النظام عن قاعدة تطابق عنوان العميل والكمية والوزن وقيمة الطلب، ثم يحسب الشحن.

## المخزون

### المستودعات

المسار:

```text
Inventory > Warehouses
```

- `Name`: اسم المستودع.
- `Code`: كود فريد للمستودع.
- `Address`: العنوان.
- `City`: المدينة.
- `Country`: البلد.
- `Is active`: تفعيل المستودع.

### حركات المخزون

المسار:

```text
Inventory > Inventory Movements
```

- `Warehouse`: المستودع.
- `Product variant`: المتغير المرتبط.
- `Type`: نوع الحركة، شراء أو بيع أو تسوية أو مرتجع.
- `Quantity`: الكمية. السالب يعني خروج مخزون.
- `Source type`: مصدر الحركة مثل Order.
- `Source ID`: رقم المصدر.
- `Metadata`: بيانات إضافية مثل رقم الطلب.

النظام يضيف حركة بيع تلقائياً عند إنشاء الطلب، ويخصم المخزون داخل transaction.

## التسويق

### Hero Slides

المسار:

```text
Marketing > Hero Slides
```

تتحكم بسلايدر الصفحة الرئيسية.

داخل كل لغة:

- `Title`: العنوان الرئيسي.
- `Eyebrow`: نص صغير فوق العنوان.
- `Subtitle`: وصف السلايد.
- `Primary button`: نص الزر الأساسي.
- `Secondary button`: نص الزر الثانوي.

حقول مشتركة:

- `Image`: صورة السلايد.
- `Primary URL`: رابط الزر الأساسي.
- `Secondary URL`: رابط الزر الثانوي.
- `Sort order`: ترتيب السلايد.
- `Starts at / Ends at`: فترة ظهور السلايد.
- `Is active`: تفعيل أو إخفاء.

### Banners

المسار:

```text
Marketing > Banners
```

تستخدم لصور تسويقية في الصفحة الرئيسية أو صفحات المتجر.

- `Title / Subtitle / Eyebrow`: النصوص حسب اللغة.
- `Primary button / Secondary button`: نصوص الأزرار.
- `Image`: صورة البانر.
- `Primary URL / Secondary URL`: روابط الأزرار.
- `Placement`: مكان الظهور مثل `home` أو `shop`.
- `Sort order`: الترتيب.
- `Starts at / Ends at`: فترة الظهور.
- `Is active`: تفعيل.

### Flash Offers

المسار:

```text
Marketing > Flash Offers
```

- `Title / Description`: عنوان ووصف العرض حسب اللغة.
- `Slug`: يولد تلقائياً من العنوان ولا يحتاج إدخال يدوي.
- `Type`: نوع العرض مثل خصم نسبة، خصم قيمة، سعر ثابت لكمية، حزمة منتجات، شحن مجاني، أو اشتر كمية واحصل على كمية مجانية.
- `Status`: حالة العرض `draft / active / inactive / expired`.
- `Starts at / Ends at`: فترة صلاحية العرض.
- `Priority`: أولوية العرض عند وجود أكثر من عرض على نفس المنتج.
- `Discount value`: قيمة الخصم في عروض النسبة أو القيمة الثابتة.
- `Fixed price`: السعر الثابت في عروض السعر المحدد أو الحزم.
- `Max quantity / Sold quantity`: الحد الأقصى للكمية وعدد الكميات المباعة من العرض.
- `Free shipping`: يجعل الطلب بدون رسوم شحن عند تطبيق العرض.
- `Usage limit / Usage per user`: حدود الاستخدام الحالية المجهزة للتوسع.
- `Offer Products`: المنتجات الداخلة في العرض، مع الكمية والسعر الأصلي وسعر العرض وتحديد المنتج المجاني عند عروض Buy X Get Y.

ملاحظة: لا يتغير سعر المنتج الأصلي عند إنشاء العرض. السعر النهائي يحسب مؤقتاً من السيرفر أثناء العرض، ثم يعود السعر الطبيعي تلقائياً عند انتهاء العرض.

### الكوبونات

المسار:

```text
Marketing > Coupons
```

- `Code`: كود الكوبون مثل `WELCOME10`.
- `Type`: نوع الخصم، ثابت أو نسبة.
- `Value`: قيمة الخصم.
- `Minimum order amount`: أقل قيمة طلب لاستخدام الكوبون.
- `Starts at`: تاريخ البداية.
- `Expires at`: تاريخ الانتهاء.
- `Usage limit`: عدد مرات الاستخدام.
- `Used count`: عدد الاستخدامات الحالي، للقراءة فقط غالباً.
- `Is active`: تفعيل الكوبون.

عند checkout يتحقق النظام من صلاحية الكوبون ويقفل السجل أثناء الاستخدام لتجنب تجاوز حد الاستخدام.

### المراجعات

المسار:

```text
Marketing > Reviews
```

- `Product`: المنتج.
- `User`: العميل.
- `Rating`: التقييم من 1 إلى 5.
- `Title`: عنوان المراجعة.
- `Comment`: نص المراجعة.
- `Is published`: نشر أو إخفاء.

المراجعات المنشورة تظهر في الواجهة وتؤثر على تقييم المنتج.

## العملاء والشراكات

### المستخدمون

المسار:

```text
Customers > Users
```

- `Name`: اسم المستخدم.
- `Email`: البريد.
- `Mobile`: رقم الهاتف.
- `Type`: نوع الحساب.
- `Password`: كلمة المرور. مطلوبة عند الإنشاء، اختيارية عند التعديل.
- `Status`: تفعيل أو تعطيل الحساب.
- `Roles`: أدوار الصلاحيات.

أنواع الحسابات:

- `Super Admin`: صلاحيات كاملة.
- `Admin`: إدارة عامة.
- `Manager`: إدارة عمليات ومنتجات.
- `Customer`: عميل عادي.
- `Wholesale Customer`: تاجر جملة.

### طلبات الانضمام كتاجر جملة

المسار:

```text
Customers > Wholesale Applications
```

الطلب يأتي من صفحة:

```text
/join-us
```

#### Application Details

- `Full Name`: اسم مقدم الطلب.
- `Email`: البريد الذي سيستخدم للحساب.
- `Phone`: رقم الهاتف.
- `WhatsApp`: رقم واتساب.
- `Business Name`: اسم المحل أو الشركة.
- `Business Type`: نوع النشاط التجاري.
- `City`: المدينة.
- `Address / Location`: العنوان.
- `Additional Notes`: ملاحظات العميل.

#### Review

- `Status`: pending أو approved أو rejected.
- `Admin Notes`: ملاحظات داخلية أو سبب الرفض.
- `Linked User`: ربط الطلب بمستخدم موجود.
- `Reviewed By`: الأدمن الذي راجع الطلب.
- `Reviewed At`: وقت المراجعة.

#### قبول الطلب

عند الضغط على Approve:

1. إذا لم يكن هناك مستخدم بنفس البريد، ينشئ النظام حساباً جديداً.
2. يعطي المستخدم نوع `wholesale_customer`.
3. يضيف Role باسم `Wholesale Customer`.
4. يرسل بريد Mailtrap يحتوي رابط إنشاء كلمة المرور.
5. بعد تعيين كلمة المرور يستطيع التاجر تسجيل الدخول ورؤية أسعار الجملة.

إذا كان هناك مستخدم بنفس البريد، يجب ربط الطلب بالمستخدم أولاً ثم الموافقة، لتجنب ترقية حساب بالخطأ.

## الإعدادات

### العملات

المسار:

```text
Settings > Currencies
```

- `Code`: كود العملة مثل USD.
- `Symbol`: رمز العملة.
- `Name`: اسم العملة.
- `Rate`: سعر الصرف مقابل العملة الأساسية.
- `Is default`: العملة الرئيسية.
- `Status`: تفعيل العملة.

مهم: العملة الأساسية يجب أن يكون `Rate = 1`. في البيانات التجريبية الدولار هو العملة الأساسية.

### Settings

المسار:

```text
Settings > Settings
```

هذا القسم للقيم العامة التي لا تحتاج شاشة مستقلة.

- `Group`: مجموعة الإعداد مثل `general`, `identity`, `contact`.
- `Key`: مفتاح برمجي مثل `store.name`.
- `Type`: نوع القيمة، نص أو رقم أو boolean أو JSON.
- `Value`: القيمة نفسها.
- `Is public`: هل يمكن استخدامها في واجهة المتجر.

الأفضل تعديل الهوية والتواصل من `Site Appearance` بدلاً من تعديل المفاتيح يدوياً.

## سيناريوهات عمل سريعة

### إضافة منتج هاتف

1. افتح `Catalog > Products`.
2. اضغط Create.
3. أدخل الاسم والوصف بالعربية والإنجليزية.
4. أدخل `slug`, `sku`, `barcode`.
5. اختر البراند مثل Samsung أو Apple.
6. اختر التصنيف `Smartphones`.
7. أدخل سعر التجزئة، المخزون، الوزن.
8. أضف صور المنتج.
9. أضف متغيرات اللون والذاكرة.
10. بعد الحفظ افتح Price Tiers وأضف شرائح الجملة.

### إنشاء عرض على منتج

1. افتح `Marketing > Flash Offers`.
2. اختر نوع العرض المناسب.
3. أدخل فترة البداية والنهاية والحالة Active.
4. أضف المنتجات داخل `Offer Products`.
5. أدخل الخصم أو السعر الثابت حسب نوع العرض.
6. يظهر العرض في صفحة العروض وبطاقة المنتج إذا كان فعالاً وضمن الفترة والكمية المتاحة.

### قبول تاجر جملة

1. افتح `Customers > Wholesale Applications`.
2. راجع بيانات الطلب.
3. اضغط Approve.
4. يصل بريد للتاجر عبر Mailtrap.
5. التاجر ينشئ كلمة مرور.
6. بعد الدخول يرى شرائح أسعار الجملة.

### معالجة طلب يدوي الدفع

1. العميل يختار Manual أو Bank Transfer ويرفع صورة الإيصال.
2. افتح `Sales > Payments`.
3. راجع الإيصال والمبلغ.
4. غيّر الدفع إلى Paid أو استخدم Action من الطلب.
5. افتح الطلب وانقله إلى Processing ثم Shipped.

### إعداد شحن حسب مدينة ووزن

1. أنشئ Shipping Zone للدولة والمدينة والبلدة.
2. أنشئ Shipping Method نوع `rule_based`.
3. أنشئ Shipping Rule واختر method وzone.
4. اختر `By Weight`.
5. أدخل `Cost` و`Cost per kg`.
6. عند checkout يحسب النظام التكلفة من وزن المنتجات وعنوان العميل.
