# دليل استخدام لوحة إدارة متجر البيك

هذا الدليل مخصص للأدمن لتجربة وإدارة بيانات المتجر من لوحة Filament.

للتوثيق التفصيلي الكامل لكل إدارة وحقول الإضافة والتعديل وآلية العمل، راجع:

```text
docs/ADMIN_STORE_DOCUMENTATION.md
```

ويمكن فتحه من المتصفح بعد تسجيل دخول الأدمن عبر:

```text
http://127.0.0.1:8000/admin/documentation
```

## الدخول

افتح:

```text
http://127.0.0.1:8000/admin/login
```

## واجهة المتجر للزائر والعميل

تم تجهيز Web Routes للمتجر بدون API، وهي تعمل عبر Controllers وBlade views قابلة للاستبدال لاحقاً بتصميم احترافي.

المسارات العامة:

```text
/                     الصفحة الرئيسية للمتجر
/shop                 صفحة المنتجات مع بحث وفلاتر
/products/{slug}      تفاصيل المنتج
/brands               العلامات التجارية
/brands/{slug}        منتجات علامة تجارية
/offers               العروض والكوبونات
/contact              صفحة التواصل
```

مسارات حساب العميل:

```text
/account/login        تسجيل دخول العميل
/account/register     إنشاء حساب عميل
/account              لوحة حساب العميل
/account/profile      الملف الشخصي
/account/addresses    عناوين العميل
/account/orders       طلبات العميل
/account/orders/{id}  تفاصيل طلب العميل
```

مسارات عمليات الشراء:

```text
/cart                 السلة
/checkout             إتمام الطلب
/wishlist             المفضلة
```

ملاحظات:

- مسار `/login` لا يزال موجهاً إلى `/admin/login` حتى يبقى دخول الأدمن واضحاً.
- دخول العميل يتم من `/account/login`.
- صفحات المتجر الحالية تشغيلية وبسيطة، وليست التصميم النهائي للواجهة.
- جميع عمليات السلة والدفع والطلبات محمية بـ `auth` وتتحقق من ملكية بيانات العميل.

## تغيير اللغة

من أعلى لوحة التحكم يوجد مفتاح:

- English
- العربية

عند اختيار العربية تتحول اللوحة إلى RTL وتظهر التسميات العربية. عند اختيار English تعود اللوحة إلى LTR.

## لوحة التحكم

تحتوي الصفحة الرئيسية على:

- إجمالي المبيعات.
- عدد الطلبات.
- عدد المنتجات.
- عدد العملاء.
- جدول آخر الطلبات.

## الكتالوج

### العلامات التجارية

المسار:

```text
Catalog > Brands
```

الاستخدام:

- إضافة علامة تجارية جديدة.
- تعديل الاسم والوصف والشعار.
- حقل Logo يرفع صورة فعلية ويحفظ مسارها داخل التخزين العام، وليس رابطاً نصياً.
- حذف العلامات غير المستخدمة.

### التصنيفات

المسار:

```text
Catalog > Categories
```

الاستخدام:

- إنشاء تصنيف رئيسي.
- إنشاء تصنيف فرعي عبر اختيار Parent Category.
- تعطيل تصنيف عبر حقل الحالة.

### الموردون

المسار:

```text
Catalog > Suppliers
```

الاستخدام:

- إدارة موردي المنتجات.
- حفظ بيانات التواصل والعنوان.
- تفعيل أو تعطيل المورد.

### المنتجات

المسار:

```text
Catalog > Products
```

الاستخدام:

- إضافة منتج جديد.
- ربط المنتج بعلامة تجارية وتصنيف ومورد.
- Barcode حقل نصي مقصود، لأن الباركود قد يبدأ بصفر وقد يحتوي رموزاً أو صيغة EAN/UPC/QR، ولو كان رقماً قد نخسر الأصفار الأولى.
- تحديد سعر التجزئة وسعر الجملة.
- ضبط أقل كمية لتفعيل سعر الجملة.
- إضافة صور متعددة.
- إضافة Variants مثل اللون والحجم والنكهة.
- ضبط المخزون لكل Variant.
- تحديد المنتج كمميز.

## المبيعات

### الطلبات

المسار:

```text
Sales > Orders
```

الحالات المدعومة:

- Pending
- Paid
- Processing
- Shipped
- Delivered
- Cancelled
- Refunded

الإجراءات المتاحة من جدول الطلبات:

- Invoice: تحميل فاتورة PDF.
- Mark Paid: تحويل الطلب إلى مدفوع.
- Process: نقل الطلب إلى المعالجة.
- Ship: إدخال رقم تتبع ونقل الطلب إلى الشحن.
- Deliver: إنهاء الطلب كمسلم.
- Cancel: إلغاء الطلب.

### المدفوعات

المسار:

```text
Sales > Payments
```

الاستخدام:

- مراجعة حالة الدفع لكل طلب.
- متابعة قيمة الدفع ومرجع العملية.
- تعديل حالة الدفع عند الحاجة.

### طرق الدفع

المسار:

```text
Sales > Payment Methods
```

طرق الدفع التجريبية:

- Cash on Delivery
- Bank Transfer
- Manual Payment

يمكن تفعيل أو تعطيل أي طريقة دفع.

إعداد الدفع اليدوي:

- Name: اسم طريقة الدفع الذي يظهر للعميل.
- Image: صورة أو شعار طريقة الدفع.
- Wallet Link: رابط المحفظة، رقم الحساب، رقم الهاتف، أو IBAN الذي سيدفع إليه العميل.
- Barcode Image: صورة QR أو باركود المحفظة.
- Fee: رسوم إضافية لطريقة الدفع إن وجدت.

آلية العمل:

- العميل يختار طريقة الدفع في `/checkout`.
- إذا كانت الطريقة Bank Transfer أو Manual تظهر له معلومات المحفظة والباركود.
- العميل يرفع صورة إشعار الدفع قبل إنشاء الطلب.
- الدفع ينتقل إلى `Manual Review`.
- الأدمن يراجع الإيصال من `Sales > Payments`.
- عند الضغط على Approve Payment يتحول الدفع إلى Paid، وإذا كان الطلب Pending ينتقل إلى Paid.
- بعد ذلك ينتقل الأدمن بالطلب إلى Processing ثم Shipped من `Sales > Orders`.

### الشحن

المسارات:

```text
Sales > Cities
Sales > Shipping Carriers
Sales > Shipping Rates
```

الاستخدام:

- إنشاء مدينة فعالة تظهر في العناوين وcheckout.
- إنشاء شركة شحن فعالة.
- ربط شركة الشحن بالمدينة عبر سعر شحن.
- حساب الشحن حسب المدينة، شركة الشحن، ووزن المنتجات القابلة للشحن.

إعداد المدن:

- Country: البلد.
- Name: اسم المدينة بالعربية والإنجليزية.
- Code: كود اختياري.
- Is Active: هل تظهر المدينة للعميل.

إعداد شركات الشحن:

- Name: اسم الشركة.
- Logo: شعار الشركة، يحفظ في التخزين العام.
- Tracking URL: رابط تتبع اختياري.
- Status: active أو inactive.

إعداد أسعار الشحن:

- Shipping Carrier: شركة الشحن.
- City: المدينة التي تغطيها الشركة.
- Base Cost: التكلفة الأساسية.
- Cost Per Kg: تكلفة كل كيلو من الوزن القابل للفوترة.
- Min/Max Weight: شروط حسب وزن الطلب.
- Free Shipping Threshold: حد قيمة الطلب للشحن المجاني.
- Estimated Delivery Time: وقت التسليم المتوقع.
- Remote Area Fee: رسوم منطقة بعيدة.

عند checkout يعرض النظام شركات الشحن المتاحة للمدينة المختارة فقط، ثم يعيد Backend حساب التكلفة قبل إنشاء الطلب. المنتج الذي عليه شحن مجاني لا يصفر شحن السلة كاملة، بل يستثنى وزنه فقط إلا إذا كان العرض مخصصاً لشحن الطلب بالكامل.

## الإعدادات

### العملات

المسار:

```text
Settings > Currencies
```

العملات التجريبية:

- USD
- TRY
- SYP

آلية العمل:

- الدولار USD هو العملة الرئيسية ويجب أن يكون Rate = 1.
- كل عملة أخرى يكون سعرها بالنسبة للدولار. مثال: إذا كان 1 USD = 30 TRY فضع Rate = 30.
- عند تفعيل Is Default لعملة معينة يقوم النظام تلقائياً بإلغاء العملة الرئيسية السابقة حتى تبقى عملة رئيسية واحدة فقط.
- Status يحدد هل العملة متاحة للاستخدام أم لا.

### الإعدادات العامة

المسار:

```text
Settings > Settings
```

الاستخدام:

- اسم المتجر.
- اللغة الافتراضية.
- بريد الدعم.
- إعدادات تنبيهات المخزون.

الفكرة من Settings:

- هذا القسم مخصص للقيم العامة التي لا تستحق جدولاً مستقلاً، مثل اسم المتجر، بريد الدعم، الحد الأدنى لتنبيه المخزون، وروابط التواصل.
- Group يستخدم لتنظيم الإعدادات مثل general أو localization أو inventory.
- Key هو اسم الإعداد البرمجي مثل store.name.
- Type يوضح نوع القيمة: نص، رقم، Boolean، أو JSON.
- Value يحتوي قيمة الإعداد. في القيم متعددة اللغات يمكن استخدام JSON مثل ar و en.
- Is Public يعني أن الإعداد مسموح إظهاره مستقبلاً للواجهة العامة، أما الإعدادات الداخلية فتبقى غير عامة.

## إعادة بناء البيانات التجريبية

لإعادة إنشاء قاعدة البيانات كاملة:

```bash
php artisan migrate:fresh --seed
```

لتحديث البيانات التجريبية فقط:

```bash
php artisan db:seed --class=ShopSeeder
```

## ملاحظات مهمة

- الصور في seeder هي مسارات تجريبية، وليست ملفات صور فعلية.
- API غير مفعل حالياً بناءً على قرار المشروع.
- لوحة Filament هي واجهة الإدارة الرئيسية حالياً.
